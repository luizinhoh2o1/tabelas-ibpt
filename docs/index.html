Documentacao da API

API estatica hospedada no GitHub Pages. Os dados sao arquivos JSON comprimidos com gzip, gerados automaticamente a partir das tabelas oficiais do IBPT, conforme a Lei da Transparencia Fiscal (Lei 12.741/2012).

Todas as versoes/tabelas de cada ano estao disponiveis (1o e 2o semestre, revisoes A, B, C, etc.).

Como Funciona o Sistema

O sistema opera em 3 etapas:

  • Construcao (build): Um script TypeScript extrai os ZIPs do repositorio, le cada CSV via streaming, agrupa os registros por ano, versao/tabela, tipo e UF, converte para JSON com propriedades descritivas e comprime com gzip nivel 9. Todos os CSVs sao processados em paralelo.
  • Publicacao (deploy): O GitHub Actions executa o build automaticamente a cada push e publica os arquivos no GitHub Pages. Os .json.gz ficam acessiveis como URLs estaticas.
  • Consulta: O usuario pode usar esta pagina interativa (busca client-side) ou acessar os endpoints diretamente via fetch, cURL, Python, etc.

Sistema de Compressao

Os dados originais em JSON somam ~1.7 GB. Para viabilizar a hospedagem no GitHub Pages, todos os arquivos de dados sao comprimidos com gzip nivel 9, reduzindo para ~314 MB (82% de reducao).

  • Arquivos de dados: extensao .json.gz (comprimidos)
  • Arquivos de indice: extensao .json (sem compressao, pequenos)
  • Descompressao no browser: DecompressionStream('gzip') - API nativa (Chrome 80+, Firefox 113+, Safari 16.4+)
  • Descompressao via cURL: curl URL | gunzip
  • Descompressao via Python: gzip.decompress(resp.content)

Como Funciona a Pesquisa

A pesquisa e inteiramente client-side - nao existe backend. O fluxo e:

  1. Metadados: Ao abrir a pagina, o arquivo /api/meta.json e carregado. Ele contem a lista de anos, versoes por ano, UFs e tipos. Os filtros sao populados dinamicamente a partir desses metadados.
  2. Montagem: Ao clicar em "Consultar", a pagina monta a lista de combinacoes {ano}/{versao}/{tipo}/{uf} com base nos filtros selecionados. Filtros vazios incluem todas as opcoes daquele campo.
  3. Busca em lotes: Os arquivos .json.gz sao buscados em lotes paralelos de 8 requisicoes simultaneas. Uma barra de progresso acompanha o andamento. Cada arquivo e descomprimido com DecompressionStream e os registros sao filtrados por codigo (prefixo) e/ou descricao (substring case-insensitive).
  4. Exibicao: Os resultados sao exibidos em tabela paginada (100 registros/pagina), ordenavel por qualquer coluna. Os resultados podem ser exportados como CSV.

Dica de desempenho: Quanto mais filtros voce selecionar, mais rapida sera a consulta. Uma busca com ano + versao + tipo + UF especificos baixa apenas 1 arquivo (~190 KB).

Tipos de Dados

  • NCM Nomenclatura Comum do Mercosul - Codigos de produtos (8 digitos). ~11.000 registros por UF.
  • NBS Nomenclatura Brasileira de Servicos - Codigos de servicos (9 digitos). ~860 registros por UF.
  • LC116 Lei Complementar 116 - Codigos de servicos municipais (4 digitos). ~200 registros por UF.

Endpoints Disponiveis

Base URL:

Arquivos de dados usam extensao .json.gz (gzip). Indices usam .json (sem compressao).

  • GET
    /api/meta.json
    Metadados: anos disponiveis, todas as versoes/tabelas, tipos e UFs.
  • GET
    /api/{ano}/index.json
    Indice do ano com todas as versoes e contagem de registros. Ex: /api/2026/index.json
  • GET
    /api/{ano}/{tabela}/index.json
    Indice de uma versao/tabela especifica. Ex: /api/2026/26.1.F/index.json
  • GET
    /api/{ano}/{tabela}/{tipo}/index.json
    Indice por tipo com contagem por UF. Ex: /api/2026/26.1.F/ncm/index.json
  • GET
    /api/{ano}/{tabela}/{tipo}/{uf}.json.gz
    Dados completos (gzip). Ex: /api/2026/26.1.F/ncm/SP.json.gz

Descompressao (gzip)

// JavaScript - descomprimir com DecompressionStream
async function buscarDados(url) {
  const resposta = await fetch(url);
  const fluxo = resposta.body.pipeThrough(new DecompressionStream('gzip'));
  const texto = await new Response(fluxo).text();
  return JSON.parse(texto);
}

// Uso
const dados = await buscarDados('/api/2026/26.1.F/ncm/SP.json.gz');
dados.dados.forEach(item => {
  console.log(`${item.codigo}: Federal=${item.aliquotaNacionalFederal}%, Estadual=${item.aliquotaEstadual}%`);
});
# cURL - descomprimir com gunzip
curl -s https://SEU_USUARIO.github.io/REPO/api/2026/26.1.F/ncm/SP.json.gz | gunzip | jq '.dados | length'

# Buscar codigo especifico
curl -s https://SEU_USUARIO.github.io/REPO/api/2026/26.1.F/ncm/SP.json.gz | gunzip | jq '.dados[] | select(.codigo=="22030000")'
# Python
import requests, gzip, json

resp = requests.get('https://SEU_USUARIO.github.io/REPO/api/2026/26.1.F/nbs/MG.json.gz')
dados = json.loads(gzip.decompress(resp.content))

for item in dados['dados']:
    print(f'{item["codigo"]}: {item["descricao"]} - Federal: {item["aliquotaNacionalFederal"]}%, Estadual: {item["aliquotaEstadual"]}%')
